Testing Details

any details required to test the site, including instructions if needed, outstanding issues (like things that look as though they work but don't)

  • This was designed for large monitors only. Therefore the website will crop out on smaller devices (phones, tablets, etc...)
  • Fixed widths and position tools were used instead of percentages of the screen size. Therefore it will not change if the browser size or screen size changes.
  • No Javascript was implemented due to time constraints (was going to be used for validity checking in sign-up sheet)
  • Not all artists have their own pages, only 5 pages have these available.
  • These 5 are: Adam Lopez, Domenico Taraborrelli, Jennifer Bradstreet, Richard McIntyre and Sean O'Boyle

PMI

your group should spend some time reflecting on the process and describe here the Pluses, Minuses and Interesting aspects of your teamwork process and result

Pluses:

  • We were able to work on the project at the same time as long as we worked on different pages
  • Work was divided between the two of us so we could work on our tasks in our own time
  • Able to bounce ideas off of each other and help each other when unsure about certain choices.
  • Jason focused more on structural layout and content, while Ghazala focused more on CSS styles.

Minuses:

  • GITHUB would overwrite other collaborators work when synchronising the local repository on the desktop with the online repository if both of us were working on the same file
  • Had some creative differences in the CSS styles used at first. An agreement was reached.

Interesting:

  • HTML has the strangest commenting syntax I have ever seen.
  • Designing the page layouts using CSS was a lot more efficient than using a table format
  • Div blocks are essential for good structural layout.
  • A combination of HTML, CSS and Javascript is essential for successful websites.
